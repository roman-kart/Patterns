# Ambient Context
## Мотивация
**Цель:** Конкретезировать приложение экземпляром того класса, который необходим во время выполнения.</br>
**Суть:** Статическое свойство Instance вместо возврата конкретного класса возвращает экземпляр абстрактного класса или интерфейса.
Также появляется setter свойства, который позволяет установить нужный экземпляр singleton при старте приложения, во время смены контекста 
или во время инициализации Unit-тестов.</br>
В результате решение будет более гибким и тестируемым, в отличие от классической реализации паттерна.
## Недостатки
### Не гарантирует наличие единственного экземпляра
Данный вариант реализации не гарантирует наличие одного экзепляра, а лишь обеспечивает глобальную точку доступа к некоторой зависимости.
## Применение
Подходит для использования зависимости разными слоями приложения, делая эти слои менее зависимыми друг от друга.
При этом определенная гибкость обеспечивается за счет возможности установить глобальное состояние при старте приложения.
